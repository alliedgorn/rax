## Learnings

- **DO cloud firewalls block at network level** — no logs on the server, invisible unless you know to check the API.
- **Cloudflare 522 ≠ always firewall** — can also be wrong origin IP, SSL mode mismatch, or HTTP redirect loops.
- **Cloudflare edge propagation** — IP list changes take minutes to sync across all edge nodes. Patient waiting > frantic debugging.
- **Defense in depth** — bind to localhost + firewall + WAF. Any one layer can fail.
- **Document as you go** — Gorn asked for infra docs after the Cloudflare work. Would have been easier to document during.
