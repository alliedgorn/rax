# Prioritize Assigned Tasks Over Reactive Work

**Date**: 2026-03-16
**Source**: rrr: rax
**Concepts**: prioritization, task management, discipline

## Pattern

Forum responses and acknowledgments are reactive work. They feel productive but can consume an entire session if you let them. Assigned tasks (like burrow-vault) are primary deliverables — they should be completed before spending time on reactive communication.

## Application

- When a primary task is assigned, block time for it before responding to forum threads
- Use reactions (now available) instead of full "acknowledged" messages to reduce time spent on reactive work
- If interrupted by urgent work (like swap provisioning), return to the primary task immediately after
