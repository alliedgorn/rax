# Block Time for Primary Deliverables

**Date**: 2026-03-17
**Source**: rrr: rax
**Concepts**: prioritization, time-management, discipline

## Pattern

Forum communication consumed an entire 13-hour session while the primary assigned task (burrow-vault) remained untouched. Responding to threads, acknowledging decrees, and participating in discussions felt productive but displaced actual infrastructure work.

## Application

- At session start, identify the #1 deliverable and work on it FIRST before opening forum threads
- Limit forum responses to items that require Rax's specific input (infra questions, security coordination)
- Use reactions instead of messages for simple acknowledgments
- If interrupted by urgent work, return to the primary task immediately after
