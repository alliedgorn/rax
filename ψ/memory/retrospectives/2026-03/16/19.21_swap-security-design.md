## Honest Feedback

**1. Password in session context.** Gorn had to share his sudo password because I couldn't run privileged commands otherwise. This is a real security concern for an AI-assisted infrastructure workflow. There should be a better pattern — maybe a pre-authorized sudo for specific commands, or a helper script that Gorn runs manually for privileged operations.

**2. Forum thread sprawl is already starting.** Thread #18 became a catch-all. Thread #20 mixed security discussion with profile pages and decrees. Thread #22 mixed forum features with swap provisioning. Leonard's topic-based naming convention helps, but the discipline to create new threads per topic isn't consistent yet.

**3. Notification fatigue risk.** Every @all decree triggers every Beast. With 7 Beasts all acknowledging, that's 6 notification-driven responses per decree, most just saying "acknowledged." The forum needs a reaction/emoji system so "acknowledged" can be a thumbs-up instead of a full message.

## Lessons Learned

1. **Infrastructure shapes experience** — tunnel-style navigation in Burrow Book came from thinking about how infra architecture affects the people moving through it. Pipes aren't just plumbing.
2. **Privileged operations need a pattern** — sudo in AI sessions needs guardrails. Plan ahead for what needs root access.
3. **Swap is cheap insurance** — 4GB swapfile took 30 seconds to create and gives the system breathing room. Should be standard on any Beast-hosting machine.
