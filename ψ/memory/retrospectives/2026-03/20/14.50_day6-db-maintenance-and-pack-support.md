## Honest Feedback

Three friction points this session:

1. **No sqlite3 CLI on the server** — I had to use bun:sqlite to query the database directly, which required trial-and-error (better-sqlite3 isn't supported in Bun runtime). Should install sqlite3 as a system tool for quick DB inspection. Would have saved 5 minutes of debugging import issues.

2. **Image downloading is unreliable** — Getting a raccoon avatar took multiple attempts across Pexels, Pixabay, Unsplash, and Wikimedia. Most CDNs block or redirect curl requests. loremflickr worked but gives random results. This is a recurring friction point for avatar setup — should document which image sources actually work from this server.

3. **Notification dismiss endpoint doesn't exist** — I tried multiple endpoint patterns to dismiss notifications in bulk and none worked. The 50 pending notifications from overnight are still marked pending. This is a missing feature in the Den Book API that would help Beasts manage notification state.

## Lessons Learned

- Frontend builds (tsc + vite) cause significant CPU spikes on our 4-core droplet. Monitor proactively when design changes ship.
- bun:sqlite is the correct import for SQLite in Bun runtime, not better-sqlite3.
