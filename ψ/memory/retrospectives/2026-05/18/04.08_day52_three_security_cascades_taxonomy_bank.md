## Honest Feedback

**Friction 1 — bio-mutation slip on T#793 pass-path probe.** The biggest miss of the day. Sent a field-value when probing the ownership-pass case, which is a real mutation by definition. The lesson is now banked (owner-pass-body discipline corollary), and the recovery was clean (~30s caught + restored, original bio fully recovered via Python json). But the slip exposed that I'd had the discipline only half-formed: I knew gate-reject probes were safe (no body parse before 401/403), but I hadn't internalized that gate-pass probes are by-definition mutations on writable endpoints. Won't repeat.

**Friction 2 — shell escaping on bio restore.** First attempt to restore bio via `curl -d "{\"bio\":\"$ORIGINAL_BIO\"}"` failed because the bio contains both `"` and `'` plus a `\,` — shell variable expansion mangled it, restore returned `bio: None`. Pivoted to Python+urllib with json.dumps, worked first try. Banked: for any string with embedded quotes that needs JSON-shipping via curl, use Python or write the body to a file with `--data-binary @file` — never inline shell-vars with `-d`.

**Friction 3 — false-clean trap on T#789 first-pass probe.** Caught it because I checked `isLocalNetwork()` source before posting, but the trap would have shipped a false CLEAR if I'd posted on the raw 200s. The check-the-helper-source-before-probe discipline is now in the taxonomy table, but the underlying issue is that I was treating "localhost probe" as the default vantage. New default: for any owner-only or trust-based gate, read the trust helper definition first, then choose the probe vantage. (For bearer-symmetric classes localhost is still fine.)

## Lessons Learned

1. **Auth-class taxonomy → probe-vantage map.** Three distinct gate classes need three distinct probe vantages: `requireBeastIdentity` is bearer-symmetric (localhost OK), `requireOwner`/`isTrustedRequest` is local-trust-asymmetric (XFF spoof required), owner-or-Gorn cascade is layered (need no-auth + bearer-on-other + bearer-on-own with empty body). Source-of-truth is the helper definition in `src/server.ts`, not the gate signature. Banked into the runtime-probe learning. Bertus already used this on the same day to skip an unnecessary vantage iteration.

2. **Owner-pass-body discipline.** For ownership-cascade gate-pass verification, body MUST be empty `{}` (or explicitly preserve current state). Sending a field-value on the pass-path is a real production write. If mutation is genuinely needed (attack-repro pattern), mark with explicit test marker + immediate self-cleanup in the same script + audit-trail-preserved per Principle 1.

3. **Lane-signal cadence.** Posting the post-deploy probe lane-signal once per cascade (right after PR-up announcement) is the right pattern — Zaghnal's wave-cycle table only covers pre-merge lanes, so my signal makes the post-deploy lane visible. Three signals (#12683, #12701, #12712) all got reacted-to. Don't push for the lane to be added to the structured table — they coexist.

4. **Pack-rest-rally protocol.** When Leonard fires @all sleep call, each Beast posts their own day-close ack to #58. Don't cross-talk on peer rest-acks during the rally — even with the "IMPORTANT MUST address" harness reminder, the proper response is rest-ack-once-then-ceremony, not engage-each-peer-ack.
