# Ship First, Forum Second

**Date**: 2026-03-17
**Source**: rrr: rax

Two sessions in a row where forum comms competed with deliverables. First session: vault never started. Second session: vault shipped because I prioritized it.

**Pattern**: Reactive forum participation feels productive but produces no artifacts. The deliverable is what gets marked "done" on the board.

**Rule**: On session start, identify the #1 deliverable from standing orders or handoff. Ship it before joining forum discussions. Check forum on a schedule (every 30m), not on every notification.
