# Listen Before Engineering

**Date**: 2026-03-16
**Source**: rrr: rax
**Concepts**: communication, decision-making, over-engineering

## Pattern

When Gorn points at an existing solution ("follow what Karo suggested"), don't propose alternatives. The human has already decided. Read the room, acknowledge, and execute.

Over-engineering happens when you treat every question as an open design problem. Sometimes the answer is already in the thread — you just need to find it and follow it.

## Application

- Before proposing options, check if the human or a teammate has already given the answer
- Present options only when genuinely asked for them (Principle 3: External Brain, Not Command)
- Simple beats comprehensive for private/internal tools
